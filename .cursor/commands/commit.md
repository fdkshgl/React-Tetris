# コミットメッセージ作成（カテゴリ分類・日本語）

次の手順で、変更内容をカテゴリごとに分類し、日本語のコミットメッセージを1つ作成してください。

## 1. 変更の取得

- `git status` で変更ファイル一覧を確認する
- `git diff --staged` でステージ済みの差分を取得する（なければ `git diff` で未ステージの差分を取得する）
- 変更があるファイルのパスと、追加・削除・変更の概要を把握する

## 2. カテゴリ分類

変更を次のいずれかのカテゴリに分類する（複数該当する場合は主だったものを選ぶ、または複数行に分けて記載する）。

| カテゴリ | 用途 | 例 |
|----------|------|-----|
| **機能** | 新機能の追加 | 新画面、新API、新コンポーネント |
| **修正** | バグ修正・挙動修正 | クラッシュ修正、表示崩れ、計算ミス |
| **削除** | 機能・ファイル・依存の削除 | 使わない機能の削除、パッケージ削除 |
| **リファクタ** | 挙動を変えずに構造・コードの整理 | 名前変更、ファイル分割、型の整理 |
| **ドキュメント** | README、コメント、SECURITY 等 | 説明の追加・更新 |
| **設定** | ビルド・ツール・環境設定 | package.json、vite.config、.gitignore |
| **スタイル** | 見た目のみ（フォーマット・CSS等） | インデント、スタイル調整 |

## 3. コミットメッセージの形式

以下の形式で **1つのコミットメッセージ** を出力する。

```
[カテゴリ] 簡潔な説明（日本語・50字以内）

- 変更1の要約
- 変更2の要約
（必要に応じて箇条書きで補足）
```

- **1行目**: カテゴリを `[]` で囲み、その後に日本語で概要を1行で書く
- **2行目**: 空行
- **3行目以降**: 必要なら箇条書きで変更点を補足する（省略可）
- 複数カテゴリにまたがる場合は、主なカテゴリ1つを1行目にし、本文で他カテゴリに触れてもよい

## 4. 出力

- 上記形式のコミットメッセージをそのまま使える形で出力する
- メッセージのみを出力し、説明や前置きは不要とする（コピーして `git commit -m "..."` や `-F` で使う想定）
